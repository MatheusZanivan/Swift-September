CI/CD, code signing & provisioning (Fastlane, Xcode Cloud, GitHub Actions)

Matheus Z. Andrade | iOS Engineer

Introduction

Setting up CI/CD in iOS projects can feel daunting because of code signing and provisioning profiles. With the right tools, you can reduce manual effort, automate deployments, and ship more confidently. Today, we’ll look at how to manage signing, and how Fastlane, Xcode Cloud, and GitHub Actions help you create a smooth delivery pipeline.

Code Signing & Provisioning

Every iOS build needs a certificate and a provisioning profile. Automating this step ensures your CI system can build and distribute without manual intervention. Apple recommends using API keys for App Store Connect instead of storing Apple ID credentials.

Copied
// Example: Checking signing identity in a build script
import Foundation

let identity = ProcessInfo.processInfo.environment["CODE_SIGN_IDENTITY"] ?? "Not set"
print("Using signing identity: \\(identity)")

Fastlane

Fastlane automates repetitive tasks like code signing, building, and uploading to TestFlight. You can write lanes in Ruby, but your Swift project remains untouched. This makes it ideal for CI pipelines where consistency matters.

Copied
// Swift script calling Fastlane
import Foundation

let task = Process()
task.launchPath = "/usr/bin/env"
task.arguments = ["fastlane", "beta"] // runs lane 'beta'
try task.run()
task.waitUntilExit()

Xcode Cloud

Xcode Cloud is Apple’s native CI/CD system. It integrates directly with Xcode and App Store Connect, handling builds, tests, and distribution automatically. The benefit is less setup and first-class support for signing.

Copied
// Example: simple unit test
import XCTest

final class UserTests: XCTestCase {
    func testUserName() {
        let user = "Matheus"
        XCTAssertEqual(user, "Matheus")
    }
}

GitHub Actions

GitHub Actions is flexible and community-driven. You can configure workflows to build, test, and distribute your app across multiple environments. Signing assets should be stored in GitHub Secrets or fetched dynamically.

Copied
// Reading CI secret in Swift
if let apiKey = ProcessInfo.processInfo.environment["APP_STORE_CONNECT_API_KEY"] {
    print("API key loaded for CI: \\(apiKey.prefix(5))...")
} else {
    print("No API key found")
}

Best practices

  • Use encrypted storage like GitHub Secrets or Xcode Cloud’s keychain for sensitive assets.
  • Prefer App Store Connect API keys over Apple IDs for automation.
  • Keep CI builds reproducible by pinning dependencies.
  • Run unit tests before distributing builds.

Conclusion

CI/CD makes iOS development more predictable and efficient. By combining secure code signing with tools like Fastlane, Xcode Cloud, and GitHub Actions, you can release faster while reducing risk. The key is to automate responsibly and protect sensitive assets along the way.

Next steps

Try setting up a small workflow that builds your app on push and runs unit tests. Start with a single target, then add signing and distribution once you’re comfortable. Explore Apple’s guidance on continuous integration and signing to deepen your knowledge.
