Accessibility and localization (VoiceOver, Dynamic Type, RTL, dark mode)

Matheus Z. Andrade | iOS Engineer

Introduction

Accessibility and localization are not extras — they’re essentials. Building iOS apps that adapt to all users makes them more inclusive, future-proof, and user-centered. From VoiceOver support to dark mode, it’s our responsibility as developers to ensure every user can interact with the app comfortably. In this post, we’ll explore four key areas: VoiceOver, Dynamic Type, right-to-left (RTL) layout support, and dark mode.

VoiceOver

VoiceOver is a screen reader that helps visually impaired users navigate your app. Every interactive element should have clear accessibility labels and traits. Avoid exposing raw technical details like variable names.

Copied
let button = UIButton(type: .system)
button.setTitle("Submit", for: .normal)

// Accessibility
button.accessibilityLabel = "Submit form"
button.accessibilityTraits = .button

Dynamic Type

Dynamic Type ensures text scales with user font settings. Use UIFontMetrics and preferredFont for adaptive text that respects system accessibility preferences.

Copied
let label = UILabel()
label.numberOfLines = 0

// Dynamic Type
label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

Right-to-Left (RTL) Support

Some languages, like Arabic and Hebrew, are written right-to-left. iOS supports layout mirroring automatically if you use Auto Layout and system controls. Always test by forcing RTL in the scheme run settings.

Copied
let label = UILabel()
label.text = NSLocalizedString("welcome_message", comment: "Welcome text")

// Auto alignment with respect to layout direction
label.textAlignment = .natural

Dark Mode

Hard-coded colors don’t adapt to system appearance. Use semantic system colors that automatically adjust between light and dark modes.

Copied
let view = UIView()
view.backgroundColor = .systemBackground

let label = UILabel()
label.textColor = .label

Conclusion

Accessibility and localization are not optional. By supporting VoiceOver, Dynamic Type, RTL, and dark mode, we make apps that are usable by more people and resilient to future platform changes. These practices should be part of your development process from day one.

Next steps

Try running your app with larger font sizes, in dark mode, and in RTL languages to see how it adapts. Explore the Accessibility Inspector in Xcode to validate your UI. For further reading, check Apple’s official accessibility documentation.
